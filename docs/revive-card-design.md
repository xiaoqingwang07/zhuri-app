# 复活卡机制设计文档

## 一、核心概念

### 什么是复活卡
复活卡（Revive Card）是逐日App中的"后悔药"——用户漏打卡后，可以用复活卡补打卡，不断掉连续天数。

### 设计目标
1. **降低用户流失**：给用户第二次机会，不至于断了就放弃
2. **增加商业价值**：复活卡可以作为付费点或广告变现载体
3. **保持稀缺感**：不能太容易获得，否则失去激励效果

## 三、使用规则

### 3.1 使用时机
- **只能在漏打卡后使用**，不能预支
- 漏打卡后弹出提示："你漏掉了Day X，要用复活卡补上吗？"
- 用户选择"用复活卡补上"或"算了"
- 选择"算了"后， streak 归零，复活卡保留

### 3.2 复活卡余额
- 每个目标独立计算复活卡余额
- 用户可在"我的→复活卡"查看各目标余额
- 复活卡不会过期

### 3.3 使用限制
- 每次漏打卡最多使用1张
- 不能连续两天使用复活卡（必须有至少1天真实打卡）
- 复活卡不能跨目标使用

## 四、付费墙设计

### 4.1 触发时机
当用户的复活卡余额为0时，且漏打卡：
- 弹出提示："今天漏打卡了，连续天数归零"
- 同时展示付费选项："立即购买复活卡 ¥6"

### 4.2 购买入口
- 设置页 → "复活卡商店"
- 漏打卡弹窗 → "购买更多复活卡"
- 目标详情页 → 复活卡余额旁边 → "+"购买按钮

### 4.3 定价策略
- 锚定定价：3张¥15，让1张¥6显得便宜
- 首单优惠：首次购买3张立减¥3（¥12）
- 订阅制：¥18/月，每月自动补3张复活卡

## 五、广告变现设计

### 5.1 广告触发时机
- **主动触发**：用户点击"看广告得复活卡"
- **被动触发**：复活卡为0时，激励用户看广告

### 5.2 广告形式

#### 激励视频广告（Rewarded Video）
- 全屏视频，不可跳过
- 看完15-30秒视频自动发放1张复活卡
- 需要接入穿山甲/优量汇（国内）或 AdMob（海外）

#### 互动广告（Playable）
- "3道题答对2道得复活卡"
- 题库：与用户当前目标相关的问题
- 轻度游戏化，用户参与度高

#### 分享广告
- 引导用户分享到微信群/朋友圈
- 分享成功+截图回传=复活卡
- 防刷机制：同一链接每天最多计入3次

### 5.3 广告频率控制
- 每天最多3次广告获取复活卡
- 防止用户过度依赖广告而降低付费意愿
- 广告间隔至少30分钟

## 七、技术实现

### 7.1 数据模型

```typescript
interface ReviveCardBalance {
  [goalId: string]: number; // 每个目标的复活卡数量
}

interface ReviveCardTransaction {
  id: string;
  goalId: string;
  type: "earned" | "used" | "purchased" | "ad";
  amount: number;
  timestamp: string;
  source: string; // "signup" | "30day_streak" | "purchase" | "ad" | "share"
}
```

### 7.2 关键函数

```typescript
// 获取复活卡余额
function getBalance(goalId: string): number

// 使用复活卡
function useCard(goalId: string, missedDay: number): boolean

// 获得复活卡
function earnCard(goalId: string, source: string, amount: number): void

// 订阅复活卡变化
function subscribeBalance(goalId: string, callback: (balance: number) => void): Unsubscribe
```

### 7.3 推送通知触发
- 每天21:00未打卡时发送提醒
- 漏打卡后24小时内推送："可以用复活卡补救哦！"
